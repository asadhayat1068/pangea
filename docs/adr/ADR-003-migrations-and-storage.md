# ADR-003 — Migrations with golang-migrate; integer cents; UUID PKs

Status: Accepted (06/11/2026)

## Context
We need a repeatable, reviewable way to evolve the schema, and storage choices
that won't haunt us (money precision, ID strategy).

## Decision
- Schema changes are versioned SQL migrations under /migrations, run by
  golang-migrate. Each change has an up and a down file.
- Money is stored as BIGINT cents (price_cents). No floating point for money.
- Primary keys are UUIDs generated by Postgres gen_random_uuid() (built into
  Postgres 13+; no extension needed).
- Timestamps are TIMESTAMPTZ in UTC.
- Domain invariants are duplicated as DB CHECK constraints (defense in depth).

## Consequences
- Migrations run identically locally and in CI (we gate them in CI, Step 14).
- Reversible changes via down migrations.
- UUIDs cost a little space/index size vs bigserial, but avoid enumerable IDs
  and cross-tenant sequence coupling — worth it for a multi-tenant platform.
